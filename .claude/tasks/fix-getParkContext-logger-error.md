# תיקון שגיאת Logger בפונקציה getParkContext

## תיאור הבעיה

הפונקציה `getParkContext` במערכת ה-MCP של הפארקים הלאומיים נכשלת עם שגיאת `TypeError` בפונקציית הלוגינג.

## פרטי השגיאה

```
Error: Logger._log() got an unexpected keyword argument 'park_code'
Type: TypeError
Status Code: 500
Tool: getParkContext
```

## הוראות תיקון

### שלב 1: מיקום הבעיה
1. מצא את הקובץ שמכיל את הפונקציה `getParkContext`
2. חפש בקוד את הקריאות לפונקציית `Logger._log()`
3. זהה את השורה שמעבירה את הפרמטר `park_code`

### שלב 2: זיהוי הבעיה
הבעיה היא שהקוד מנסה להעביר פרמטר `park_code` לפונקציית הלוגינג, אבל הפונקציה לא מוגדרת לקבל פרמטר כזה.

### שלב 3: תיקונים אפשריים

#### אופציה 1: הסרת הפרמטר
```python
# במקום:
logger._log(park_code=park_code, message="Getting park context")

# שנה ל:
logger._log(message=f"Getting park context for {park_code}")
```

#### אופציה 2: שינוי שם הפרמטר
```python
# אם הלוגר מצפה לפרמטר אחר, למשל:
logger._log(parkCode=park_code, message="Getting park context")
```

#### אופציה 3: הוספת הפרמטר לחתימת הלוגר
אם הפרמטר נחוץ, הוסף אותו לחתימה של פונקציית `Logger._log()`.

### שלב 4: בדיקה
1. הרץ את הפונקציה `getParkContext` עם פרמטר חוקי (למשל: `parkCode="yose"`)
2. ודא שהפונקציה מחזירה תוצאה ללא שגיאות
3. בדוק שהלוגינג עובד כמו שצריך

### שלב 5: בדיקה נוספת
- בדוק אם יש שגיאות דומות בפונקציות אחרות
- ודא שכל הקריאות ללוגר עקביות

## תוצאה צפויה
לאחר התיקון, הפונקציה `getParkContext` צריכה להחזיר מידע מקיף על הפארק הנבחר, כולל:
- פרטי הפארק
- מזג אוויר נוכחי
- איכות אוויר (אם זמין)
- התראות פעילות

## הערות
- זו שגיאה פשוטה לתיקון אבל חשובה לתפקוד התקין של המערכת
- התיקון לא אמור להשפיע על פונקציות אחרות
- אחרי התיקון, בדוק שכל הכלים האחרים עדיין עובדים
